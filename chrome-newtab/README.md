# Memento · Chrome 新标签页 Dashboard

把 Chrome 的新标签页换成 Memento 的记录面板:开新标签 = 看今天留下的记录、一键复制 markdown 给 AI。

## 它做什么

- **记录摘要**:首页显示今天留下了多少条记录,默认展开完整的一天
- **一键喂任何 AI**:大按钮一点,今天的 markdown 进剪贴板,⌘V 粘给 Claude / ChatGPT / Gemini / Kimi / 豆包……
- **轻量回顾**:今日条目列表 / 90 天热力图 / 标签筛选
- **每日总结**:右侧 Tab 为每个记录日展示当天第一帧、Daily Review 和处理状态
- **Prompt 双轴 A × B**:选一个时间段 + 一个风格,复制时自动把 prompt 拼在 md 前面,粘到 AI 一次成事(见下方"模式")

`#TODO` 仍可作为历史记录标签,但不拥有完成状态、提醒权重或特殊视觉。Dashboard 不提供勾选、划线、未完成计数和数字徽章;记录下来本身就是这个产品要鼓励的动作。

## 模式 (Prompt 双轴)

CTA 上方读起来是一句话:**复制 [ 今天 ▼ ] 的 [ 精简 ▼ ] → AI**。两个下拉正交,任意组合,3 × 6 = 18 种。选择存在本地 (localStorage),下次开新标签自动恢复。

**A · 时间段**(决定喂几天 md):

| id | 名字 | 范围 |
|---|---|---|
| today | 今天 | 当天 md |
| week | 本周 | 过去 7 天拼合 |
| month | 本月 | 过去 30 天拼合 |

**B · 风格**(决定怎么整理):

| # | 名字 | 用在 |
|---|---|---|
| — | 不附 | 只复制纯 md,自己组织 |
| 1 | 精简 | 30 秒扫一眼 |
| 2 | 全面 | 日终整理 / 周报;**多天范围下自动加「跨天观察」节**(吃掉了原"复盘") |
| 3 | 发散思考 | 灵感多 / 想被推一把 |
| 4 | 教练 / 盲点 | 感觉在原地打转 |
| 5 | 情绪温度计 | 有非工作的独白记录 |
| 6 | 转 HTML 笔记 | 把材料整理成可归档的单文件 HTML |

风格文本一律「时间段中性」,材料开头会注明 `【时间范围:X】`,prompt 自己读。

还有一个**记忆卡片模式**不在下拉里——那是个彩蛋,你需要自己找到入口(提示:跟产品核心隐喻最贴),它也吃 A 时间段。

**闭环**:风格 6 输出一份自包含的 HTML 学习笔记(小报风,内置 CSS + TOC),把它存下来 → 拖进右侧「归档」抽屉 → 点击在沙箱页里安全预览。预览保留静态 HTML/CSS、`details/summary` 和页内锚点,但会移除归档自带脚本、事件属性和外部 URL。同名归档会自动加数字后缀,不会覆盖旧文件；多个 Memento 标签页也通过同一把 Web Lock 串行写入与删除。markdown 收集碎片 → AI 整理成 HTML → 归档库沉淀 + 复阅。

prompt 全文在 [prompts.js](prompts.js),要改、要加、要删,直接编辑那个文件即可。

## 每日总结 Tab

右侧功能导轨中,「归档」下方是「每日总结」。点开后,Dashboard 会以根目录的每日 Markdown 为日期底表,再读取其中的 `每日第一帧`、`Reviews/Daily/YYYY-MM-DD.md` 和 Review 运行状态,按日期组成日卡。即使当天还没有照片或总结,记录日也不会消失。

- 默认打开最新月份,可用月份下拉切换
- 每张日卡同时展示照片、时间、天气、总结摘要,并可原位展开完整总结
- 兼容 Daily Review 的新旧章节名;旧 `工作事项` / `TODO 清单` 会分别按 `工作与生活现场` / `行动线索` 展示
- Dashboard 会核对 Review 的 `source_hash`、当前 `prompt_hash`、`source_mock` 与完整输出合同，并区分“待总结 / 总结已更新 / 记录有更新 / 需重建 / 生成失败”
- 记录有更新、生成失败或过往仍待总结时,可复制一条补跑指令交给 Codex
- 某张图片缺失或损坏时,只影响照片区域;总结仍可阅读。总结尚未生成时,照片也照常保留
- 照片用浏览器临时 blob URL 显示;关闭抽屉或页面时立即回收,不复制原图、不上传

## 安装

> ⚠️ 前置条件:已经跑过主目录的 `install_aisecretary.sh`,`~/AISecretary/` 存在并有 `.md` 文件。

1. Chrome 地址栏访问 `chrome://extensions`
2. 右上角打开 **开发者模式**
3. 点 **加载已解压的扩展程序**,选 `~/AISecretary/.chrome-newtab/`(装机脚本已经把本目录拷到那里)
4. 开一个新标签 → 看到 Memento dashboard
5. 点 **「授权数据目录」** → 系统弹窗里选 `~/AISecretary` → 允许

目录句柄保存在浏览器本地 IndexedDB。之后开新标签会自动恢复;如果临时权限到期,通常只需点击一次确认,明确移除访问后才需要重新选择目录。若浏览器询问授权期限,选择“允许每次访问”可减少重复确认。恢复时会显示“读取授权记录 / 检查权限 / 读取文件”的当前阶段;浏览器授权记录的 IndexedDB 读写仍有 8 秒保护。权限检查、每日 Markdown、Daily Review、运行状态、Prompt、照片和归档都直接等待 Chrome 返回，不再给不可取消的 File System Access 调用套硬超时，也不使用跨标签页全局只读锁；偶发慢响应不会被误报为坏文件或截断成半页数据。主记录仍会先显示，可选数据随后补充；浏览器明确返回文件缺失等真实异常时，只降级对应文件。归档写入与删除继续使用跨标签页 Web Lock，避免同名覆盖和写入冲突。如果选定目录后只有 IndexedDB 保存失败,当前页仍会继续加载,并明确提示下次可能需重新选择。

## 隐私 / 安全

- Dashboard 核心功能**不发任何网络请求**,无远程依赖;归档预览会同时做 DOM 安全化与严格 sandbox CSP,阻断脚本 `location`、meta refresh、外链、远程资源和连接请求
- 文件夹访问通过 Chrome 原生 File System Access API,授权范围**仅限你选定的目录**
- 扩展本地读取不等于模型本地推理;启用 Codex Daily Review 时,当天文本会作为上下文交给已配置的 Codex 模型处理
- Markdown 渲染是内置的极简实现,不引入 marked.js / 任何第三方库
- 整个扩展是纯静态 HTML / CSS / JS,源码就在本目录,随时可读

## 卸载

`chrome://extensions` → 找到 "Memento" → 移除。新标签会自动恢复成 Chrome 默认。

## 命名说明

产品名是 **Memento**;内部技术目录(`~/AISecretary/`、`.chrome-newtab/`)和脚本(`install_aisecretary.sh`)沿用旧名 `AISecretary`,改名要迁移数据,得不偿失。
