# Memento · Chrome 新标签页 Dashboard

把 Chrome 的新标签页换成 Memento 的记录面板:开新标签 = 看今天留下的记录、一键复制 markdown 给 AI。

## 它做什么

- **记录摘要**:首页显示今天留下了多少条记录,默认展开完整的一天
- **一键喂任何 AI**:大按钮一点,今天的 markdown 进剪贴板,⌘V 粘给 Claude / ChatGPT / Gemini / Kimi / 豆包……
- **轻量回顾**:今日条目列表 / 90 天热力图 / 标签筛选
- **每日总结**:右侧 Tab 为每个记录日展示当天第一帧、Daily Review 和处理状态
- **Prompt 双轴 A × B**:选一个时间段 + 一个风格,复制时自动把 prompt 拼在 md 前面,粘到 AI 一次成事(见下方"模式")

`#TODO` 仍可作为历史记录标签,但不拥有完成状态、提醒权重或特殊视觉。Dashboard 不提供勾选、划线、未完成计数和数字徽章;记录下来本身就是这个产品要鼓励的动作。

## 模式 (Prompt 双轴)

CTA 上方读起来是一句话:**复制 [ 今天 ▼ ] 的 [ 精简 ▼ ] → AI**。两个下拉正交,任意组合,3 × 6 = 18 种。选择存在本地 (localStorage),下次开新标签自动恢复。

**A · 时间段**(决定喂几天 md):

| id | 名字 | 范围 |
|---|---|---|
| today | 今天 | 当天 md |
| week | 本周 | 过去 7 天拼合 |
| month | 本月 | 过去 30 天拼合 |

**B · 风格**(决定怎么整理):

| # | 名字 | 用在 |
|---|---|---|
| — | 不附 | 只复制纯 md,自己组织 |
| 1 | 精简 | 30 秒扫一眼 |
| 2 | 全面 | 日终整理 / 周报;**多天范围下自动加「跨天观察」节**(吃掉了原"复盘") |
| 3 | 发散思考 | 灵感多 / 想被推一把 |
| 4 | 教练 / 盲点 | 感觉在原地打转 |
| 5 | 情绪温度计 | 有非工作的独白记录 |
| 6 | 转 HTML 笔记 | 把材料整理成可归档的单文件 HTML |

风格文本一律「时间段中性」,材料开头会注明 `【时间范围:X】`,prompt 自己读。

还有一个**记忆卡片模式**不在下拉里——那是个彩蛋,你需要自己找到入口(提示:跟产品核心隐喻最贴),它也吃 A 时间段。

**闭环**:风格 6 输出一份自包含的 HTML 学习笔记(小报风,内置 CSS + TOC),把它存下来 → 拖进右侧「归档」抽屉 → 点击在沙箱页里安全预览。预览保留静态 HTML/CSS、`details/summary` 和页内锚点,但会移除归档自带脚本、事件属性和外部 URL。同名归档会自动加数字后缀,不会覆盖旧文件；多个 Memento 标签页也通过同一把 Web Lock 串行写入与删除。归档列表会先显示同 Tab 内存或本机元数据缓存，再在后台以共享 3 路并发核对文件；只有新增或修改过的 HTML 才读取最多 256 KiB 的标题前缀。关闭再打开抽屉不会重复扫描，点击缓存条目时也只精确读取对应的一个文件。markdown 收集碎片 → AI 整理成 HTML → 归档库沉淀 + 复阅。

prompt 全文在 [prompts.js](prompts.js),要改、要加、要删,直接编辑那个文件即可。

## 每日总结 Tab

右侧功能导轨中,「归档」下方是「每日总结」。点开后,Dashboard 会以根目录的每日 Markdown 为日期底表,再读取其中的 `每日第一帧`、`Reviews/Daily/YYYY-MM-DD.md` 和 Review 运行状态,按日期组成日卡。即使当天还没有照片或总结,记录日也不会消失。

- 默认打开最新月份,可用月份下拉切换
- 每张日卡同时展示照片、时间、天气、总结摘要,并可原位展开完整总结
- 兼容 Daily Review 的新旧章节名;旧 `工作事项` / `TODO 清单` 会分别按 `工作与生活现场` / `行动线索` 展示
- Dashboard 会核对 Review 的 `source_hash`、当前 `prompt_hash`、`source_mock` 与完整输出合同，并区分“待总结 / 总结已更新 / 记录有更新 / 需重建 / 生成失败”
- 记录有更新、生成失败或过往仍待总结时,可复制一条补跑指令交给 Codex
- 某张图片缺失或损坏时,只影响照片区域;总结仍可阅读。总结尚未生成时,照片也照常保留
- 照片先随文字卡片显示占位符，只在进入滚动区前后约 600 px 时读取；继续滚动才补载后面的照片。读取、缩略和解码共用 3 路并发。首次命中某张原图时生成约 480 px、质量 72% 的 WebP 衍生预览，原图不修改；缩略能力不可用时自动显示原图。合格 WebP 会写入扩展独立的本机 IndexedDB，后续新标签页可直接命中，不再重复读取、缩放和编码原图
- 持久缩略图按“目录 binding + 文件名 + 缩略规格”隔离，最多 96 项、合计 32 MiB、单项 512 KiB；查询最多等待 120 ms，超时立即按 miss 回退实时原图。原图和回退结果不会写入。当前 Tab 另保留最多 32 个临时 object URL，关闭再打开侧栏会复用已显示图片；`pagehide` 只释放当前页面的 object URL，不删除持久缩略图。缓存不联网、不上传
- 正常采集会为每张 asset 生成唯一文件名，并将其视为不可变资源。需要替换图片时应使用新文件名并更新 Markdown 引用；同名原位替换不属于常规写入路径，缓存命中不保证再次读取源文件校验

## 安装

> ⚠️ 前置条件:已经跑过主目录的 `install_aisecretary.sh`,`~/AISecretary/` 存在并有 `.md` 文件。

1. Chrome 地址栏访问 `chrome://extensions`
2. 右上角打开 **开发者模式**
3. 点 **加载已解压的扩展程序**,选 `~/AISecretary/.chrome-newtab/`(装机脚本已经把本目录拷到那里)
4. 开一个新标签 → 看到 Memento dashboard
5. 点 **「授权数据目录」** → 系统弹窗里选 `~/AISecretary` → 允许

目录句柄保存在浏览器本地 IndexedDB。之后开新标签会自动恢复；如果临时权限到期，通常只需点击一次确认，明确移除访问后才需要重新选择目录。若浏览器询问授权期限，选择“允许每次访问”可减少重复确认。浏览器授权记录的 IndexedDB 读写仍有 8 秒保护；权限检查和 File System Access 调用不套无法真正取消的硬超时。

每次新建或重新加载 Memento 标签页、且目录权限为 `granted` 时，会触发一次核对；关闭右侧抽屉不会触发，页面也没有定时重扫。暖启动会先校验并提交上次的完整快照；有效快照的首个记录画面不再闪过空 waiting，而且实时读取要等它获得一次绘制机会后才能开始。缓存校验若在 250 ms 内没有结论，实时读取会自行开始，迟到缓存只补历史且显示前重新核对权限。

随后每个 Tab 都在完整历史 Web Lock 之外直接 `getFileHandle()` 点读当天的单个 Markdown，不先遍历根目录；新增记录会立即替换缓存中的今天。若 Chrome 的今日点读仍在 pending，已显示的上次完整记录仍可通过“复制当前显示的…”立即使用，剪贴板会明确标注数据仍在核对；点读成功后按钮自动升级为普通“复制今天”。点读完成后才开始后台历史核对：leader 固定 4 路扫描根日报，并复用刚刚读过的 today 文件；follower 不做全量扫描，但仍能独立得到最新的今天。一个慢历史请求只占一路，其他三路继续；周 / 月在完整核对前也允许复制当前显示版本并携带状态标注，完整后自动去掉。Web Locks 与 BroadcastChannel 只协调历史扫描和完整快照发布，不阻塞今日点读；部分扫描永远不会覆盖上次完整快照。

目录句柄、根目录日报快照、轻量归档索引与 Daily Review 快照继续共用 `aisecretary` 本机 IndexedDB。日报快照只保存 `YYYY-MM-DD.md` 的完整原始字节，总上限 5 MiB；归档索引只保存文件名、标题和修改时间，不保存归档 HTML 正文。Daily Review 快照保存已经读取过的总结原文、运行状态、Prompt 哈希和上次核对过的日报哈希，总上限同样为 5 MiB，并始终绑定当前授权的数据目录。启动时这些数据在同一个只读事务中恢复，因此归档数量和已有总结会与记录同一帧出现，不需要先重新扫描总结目录。打开「每日总结」后只核对所选月份；每个文件读完便独立更新对应日卡，单个慢文件不会遮住其他已经可读的总结。照片缩略图使用独立的 `memento-photo-thumbnails` IndexedDB，避免缩略规格升级影响目录授权恢复；只保存 480 px WebP 衍生图，不保存原图。所有缓存均不联网，切换数据目录时会原子清除旧目录缓存；清除扩展站点数据或移除扩展也会清除缓存。

日报缓存和照片缩略图都使用目录 binding 隔离。目录切换、明确撤权、handle 身份不一致、缓存损坏或未来 schema 会 fail-closed；旧任务用 binding token CAS，不能把旧目录正文写回来。缩略图再叠加资产文件名与规格键（当前为 `w480-webp-q72-v1`），不同目录、文件名或未来规格不会串用；持久读取超过 120 ms 时直接走实时原图。另一个标签页切换目录时，旧页面会立即停用；复制前还会再次核对当前保存的目录。

Obsidian 可以继续把同一个目录当作 Vault。外部编辑刚好发生在 Chrome `getFile()` 与正文读取之间时，Chrome 会让旧 File 快照失效；Memento 仅对该文件重新获取并重试一次，不整轮重扫。目录选择提交与归档写入 / 删除共用同一把跨标签页 Web Lock；归档会在锁内再次核对当前保存的目录，避免切目录瞬间误写旧目录。如果选定目录后只有 IndexedDB 保存失败，当前页仍会继续实时加载，并明确提示下次可能需重新选择。

## 隐私 / 安全

- Dashboard 核心功能**不发任何网络请求**,无远程依赖;归档预览会同时做 DOM 安全化与严格 sandbox CSP,阻断脚本 `location`、meta refresh、外链、远程资源和连接请求
- 文件夹访问通过 Chrome 原生 File System Access API,授权范围**仅限你选定的目录**
- 快速启动缓存是每日 Markdown 的本机完整副本（上限 5 MiB）；未获得当前目录读取权限时不展示，明确撤权或切换目录时会失效
- 照片缓存只保存独立本机 IndexedDB 中的 480 px WebP 衍生缩略图（96 项 / 32 MiB / 单项 512 KiB）；原图仍只在用户选择的数据目录中
- 扩展本地读取不等于模型本地推理;启用 Codex Daily Review 时,当天文本会作为上下文交给已配置的 Codex 模型处理
- Markdown 渲染是内置的极简实现,不引入 marked.js / 任何第三方库
- 整个扩展是纯静态 HTML / CSS / JS,源码就在本目录,随时可读

## 卸载

`chrome://extensions` → 找到 "Memento" → 移除。新标签会自动恢复成 Chrome 默认。

## 命名说明

产品名是 **Memento**;内部技术目录(`~/AISecretary/`、`.chrome-newtab/`)和脚本(`install_aisecretary.sh`)沿用旧名 `AISecretary`,改名要迁移数据,得不偿失。
